# 🚀 开机启动 (Startup)

**开机启动** 功能允许您设置在 Termux-X 应用启动时自动执行特定的命令或脚本。这对于需要自动运行的服务（如 Web 服务器、定时任务等）非常有用。

## 🚀 功能说明

*   **一键开关**：侧边栏提供了一个便捷的开关按钮，用于启用或禁用开机启动功能。
*   **状态指示**：
    *   **开机启动(开)**：功能已启用，下次启动 App 时将执行预设命令。
    *   **开机启动(关)**：功能已禁用。

## ⚙️ 配置启动命令

要自定义启动时执行的命令，通常需要配合 **命令定义** 或 **配置文件** 使用：

1.  **默认行为**：Termux-X 可能会查找特定的启动脚本（如 `~/.termux/boot/` 目录下的脚本，需配合 Termux:Boot 插件），或者执行内部定义的初始化命令。
2.  **自定义命令**：您可以通过修改启动脚本（如 `.bashrc` 或 `zshrc`）来实现类似效果，但此开关主要控制 App 级别的自动运行逻辑。

## 📝 使用方法

1.  打开侧边栏菜单。
2.  找到 **开机启动** 按钮。
3.  点击按钮切换状态：
    *   如果是 **关**，点击后变为 **开**，并提示“已打开开机启动命令”。
    *   如果是 **开**，点击后变为 **关**，并提示“已关闭开机启动命令”。
4.  更改通常需要 **重启应用** 才能完全生效。

## ⚠️ 注意事项

*   如果您的启动命令包含死循环或阻塞操作，可能会导致 Termux-X 启动卡死。如果遇到这种情况，请尝试进入 **故障安全模式 (Failsafe Session)** 进行修复。
